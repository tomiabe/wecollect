# Page Blueprints — What Goes Where

These are content blueprints (section-level) so you can design and build consistently.

## Global components

- Header: nav + primary CTA (Book a demo)
- Footer: contact, legal, social, secondary links
- Proof modules: logo strip, metrics strip, testimonial (real), security badges (if valid)

## Key pages

### Home

- Hero + category statement + 2 CTAs
- Credibility strip (logos + metrics)
- Offerings (4)
- Tool-only vs Tool + People (comparison)
- Integrity (how you prove data quality)
- Solutions (4)
- Case studies (2–4)
- Engagement process
- Final CTA

### Offerings pages (Data collection / M&E / Validation / Field reporting)

- Offer definition + outcomes
- Who it’s for
- Capabilities (mapped from the feature PDF, not generic)
- Proof (how it works, examples, mini-case)
- CTA: Book a demo / Request pilot

### Platform overview

- Offline-first architecture
- Validation/integrity layers
- Reporting/export + integrations
- Security posture (only claims you can defend)

### Case study template

- Context (sector, geography)
- Challenge
- Approach (tool-only vs tool+people)
- Scale (agents, locations, timeframe)
- Integrity measures used
- Outcomes (metrics)
