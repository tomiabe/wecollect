# Content & Proof Plan — What We Need to Collect

The new positioning (“infrastructure”) requires proof. This doc lists what to gather before design locks.

## Proof inventory (needed)

- Case studies (at least 3) with:
  - scope, scale, geography
  - data integrity methods used
  - measurable outcomes
- “Tool + people” operations story:
  - agent vetting/training
  - supervision + QA loops
  - coverage map / state coverage
- Security & privacy:
  - encryption details (at rest/in transit, device)
  - access controls, audit logs
  - NDPR / GDPR posture (only if true)
- Product proof:
  - screenshots of admin console, validation, reporting dashboards
  - short demo video (optional)

## Content fixes (must-do)

- Remove placeholder/lorem copy anywhere it exists.
- Replace generic testimonials with real quotes:
  - name, role, org (or at least role + org type if anonymized)

## Asset placement rules

- Current site screenshots: `deliverables/assets/current-site/`
- New product screenshots: `deliverables/assets/product/`
- Case study images: `deliverables/assets/case-studies/<case-name>/`
- Logos: `deliverables/assets/logos/`
