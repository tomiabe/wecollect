Heuristic Evaluation — Current Site

Five critical
failures.

The current site doesn't match what WeCollect actually is — or what enterprise and government buyers need to see before they trust you with their data.

01
Wrong positioning

Current hero: "Your all-in-one Tool for Field Project Management." That reads like task software — Trello with GPS. The CEO needs to own the category of infrastructure for field data operations. These are completely different buyer conversations.

02
No offering structure

Navigation reads: Products / Use Cases / Projects / Download. None of this maps to the four offerings (collection, M&E, validation, reporting) or to how buyers engage (tool-only vs tool + people). Buyers can't find what they're looking for because the site isn't organized around them.

03
Credibility gaps

Placeholder copy on live pages. Generic testimonials. Zero visibility into how data integrity actually works. Enterprise and government buyers are choosing a data partner — they're asking "how do I know the data is clean?" That question isn't answered anywhere on the current site.

04
Four CTAs, zero funnel

"Get a Quote" vs "Get started for free" vs "Book a demo" vs "Try it for free" — four different buying signals on the same pages. Each implies a different kind of buyer and a different engagement model. The result: no one knows what to do next.

05
Design system drift

The brand guide defines #4747D6 + #1E1E1E as the core palette. The live site shows inconsistent gradients, mixed type treatments, and components that feel template-born, not designed with intent. For a company positioning as "infrastructure-grade," the visual experience doesn't match the ambition.

How Buyers Engage — Packaging

Two models.
One platform.

This is the clearest decision a buyer makes — and it's currently invisible on the site. It must be front and center, immediately understandable.

Tool only

For teams who have

You bring the enumerators. WeCollect brings the platform — offline capture, validation, reporting, project management, and agent payments infrastructure.

  • Full platform access
  • Offline + GIS capabilities
  • Validation + integrity layers
  • Admin + reporting dashboard
  • Agent payment automation
  • API integrations

Best for organizations with existing field teams who need better data infrastructure — not more logistics.

Tool + People

End-to-end field
research execution.

WeCollect provides the platform AND the field force — pre-vetted, locally-familiar agents across all 36 Nigerian states. You get data and reporting, not logistics headaches.

  • Everything in Tool only
  • 300+ LGAs covered
  • Pre-vetted agent network
  • QA loops + field supervision
  • Reporting cadence included
  • Instant agent payouts via Monify

Best for research firms, NGOs, government agencies, and enterprises without in-house field capacity — or those scaling beyond their current team.

The Integrity Layer — Trust

Data you can prove
is clean.

"Infrastructure" positioning creates a higher proof bar. The site must show exactly how WeCollect validates data — mechanism by mechanism — not just claim it.

Geofenced validation

Data entry only accepted from verified geographic locations. Wrong coordinates = flagged record. No exceptions.

Rules at point of capture

In-built validation flags bad entries in real-time — before data is ever synced to the server.

Auditor loops

Structured QA cycles with dedicated auditors reviewing submissions against field protocols — before results are finalized.

AI clean-room

Automated real-time scrubbing for duplicates, flat-lining responses, and AI-detected anomalies — before data reaches any report.

End-to-end encryption

100% encrypted from device capture to cloud storage. GDPR and NDPR compliant, with continuous audit logging and breach alerting.

PII redaction engine

Automated masking of names and phone numbers at point of export. Privacy built into the workflow — not bolted on after the fact.

Content Requirements — Proof Plan

What needs to exist
before this launches.

Positioning as "infrastructure" creates a higher credibility bar. The website must show how you prove data quality and how you execute in the field — not just list what features exist.

Must-have content (gather before design starts)

  • → 3 case studies: scope + scale + integrity methods used + measurable outcome
  • → Tool + People operating model documented: vetting process, QA loops, supervision structure, agent training
  • → Coverage evidence: map or LGA-level breakdown of 300/774 coverage across states
  • → Security and privacy claims verified: NDPR/GDPR posture, encryption architecture, storage compliance
  • → Real client testimonials: named, titled, and outcome-referenced — not generic praise
  • → Product screenshots: admin dashboard, validation layer, reporting output, agent app

Must-fix before anything ships (current site)

  • ✗ Remove all placeholder / lorem copy from live pages — this is an immediate credibility issue
  • ✗ Replace generic testimonials with real, attributed quotes from named clients
  • ✗ Convert project cards into structured case studies (use the template above)
  • ✗ Unify all CTAs to one primary enterprise path: Book a demo
  • ✗ Remove "Download" from primary navigation — move to Resources in footer
  • ✗ Align visual design to brand guide — remove inconsistent gradients and template-style components

Execution — Delivery Plan

End-to-end.
Phase by phase.

A realistic engagement model from alignment to launch. Each phase has clear deliverables and dependencies — so no phase blocks the next.

Phase 0
Alignment
1–3 days

Confirm primary audience, conversion goal, sitemap, messaging hierarchy, and tech stack. Lock the brief before anything is built.

Phase 1
UX + Content Strategy
3–7 days

Heuristic audit, page blueprints, proof and content plan, copy framework, and component inventory.

Phase 2
Design + Build
4–6 weeks

A single integrated sprint — each page is designed and shipped iteratively. Component library, responsive variants, analytics events, SEO, and performance pass all within this phase.

Phase 3
Launch + Iterate
1–2 weeks

QA, content finalization, launch, conversion monitoring, and CTA optimisation based on early data.
